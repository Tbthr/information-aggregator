Three changes needed in src/adapters/website.ts:

### CHANGE 1: Track discard count
Add a `let discardCount = 0;` variable at the top of parseWebsiteItems() function, after the cutoffTime calculation (after line 75).

### CHANGE 2: Add discard logging when item is outside 24h window
In parseWebsiteItems(), find the code block at lines 82-87:
```typescript
if (publishedTime) {
  const parsed = new Date(publishedTime);
  if (!isNaN(parsed.getTime()) && parsed >= cutoffTime) {
    publishedAt = parsed.toISOString();
  }
}
```

Replace it with:
```typescript
if (publishedTime) {
  const parsed = new Date(publishedTime);
  if (!isNaN(parsed.getTime())) {
    if (parsed >= cutoffTime) {
      publishedAt = parsed.toISOString();
    } else {
      // Item outside 24h window — log and discard
      logger.warn("Discarding item outside 24h window", {
        sourceId,
        sourceType: "website",
        title: title || "unknown",
        url,
        rawTime: publishedTime,
        discardReason: `published at ${parsed.toISOString()} is before cutoff ${cutoffTime.toISOString()}`,
      });
      discardCount++;
      return []; // Discard this item
    }
  }
}
```

### CHANGE 3: Add top-level author and content fields to RawItem
In parseWebsiteItems(), find the RawItem construction at lines 99-106:
```typescript
const item: RawItem = {
  id: `${sourceId}-${Date.now()}`,
  sourceId,
  title: title || extractTextContent(html).substring(0, 100),
  url,
  fetchedAt: new Date().toISOString(),
  metadataJson,
};
```

Change it to:
```typescript
const item: RawItem = {
  id: `${sourceId}-${Date.now()}`,
  sourceId,
  title: title || extractTextContent(html).substring(0, 100),
  url,
  author: author,           // D-06: top-level author field
  content: content,         // D-07: top-level content field
  fetchedAt: new Date().toISOString(),
  metadataJson,
};
```

### CHANGE 4: Add discard summary after collection
In parseWebsiteItems(), after the items array is populated and before returning, add the discard summary. Find the return statement at line 116 (`return [item];`) and the lines before it. Modify to:

```typescript
  // Log discard summary per source (D-04, D-05)
  logger.info("Source fetch completed", {
    sourceId,
    sourceType: "website",
    fetched: items.length,
    discarded: discardCount,
    discardRate: items.length + discardCount > 0
      ? `${((discardCount / (items.length + discardCount)) * 100).toFixed(1)}%`
      : "0%",
  });

  return items;
```

Note: Since this function returns a single item (not array), we need to restructure slightly. The discardCount must be tracked, but for a single-item website fetch, the discard summary should still be logged. If the item is discarded (outside window), we return an empty array but still log the summary with discarded: 1.
