    Update `src/ai/config/load.ts` to support `AI_DEFAULT_MODEL` as a fallback/default model override.

    **Current behavior:**
    - `ANTHROPIC_MODEL` sets the model for Anthropic provider
    - `GEMINI_MODEL` sets the model for Gemini provider
    - `OPENAI_MODEL` sets the model for OpenAI provider

    **D-15 intent:** `AI_DEFAULT_MODEL` provides a cross-provider default model string that can override individual provider models, useful for pinning the same model version across environments.

    **Implementation:**

    In `buildProviderConfig()` function, after reading `modelEnv`, check if `AI_DEFAULT_MODEL` is set and use it as the model if `modelEnv` is not set:

    ```typescript
    // D-15: AI_DEFAULT_MODEL env var support for deterministic clustering
    const defaultModel = process.env.AI_DEFAULT_MODEL?.trim()
    const model = modelEnv?.trim() || defaultModel
    ```

    This allows:
    - `AI_DEFAULT_MODEL=claude-3-5-sonnet-20241022` to set default for all providers
    - Provider-specific `ANTHROPIC_MODEL=...` overrides the default if set

    Read first: src/ai/config/load.ts (lines 70-88, buildProviderConfig function)
