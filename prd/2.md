# Web 重设计方案 v1：编辑部式日报首页 + 双维信息工作台

## Summary

Web 端重设计采用“编辑部式”视觉方向，首页定位为“今日封面”，不是传统后台列表。整体体验要让用户一打开就知道三件事：今天最值得看什么、为什么值得看、接下来该按主题还是按来源继续钻取。

设计原则已锁定：

- 视觉气质：编辑部式，强调阅读可信度、层级和节奏感
- 首页编排：封面故事
- 信息主轴：日报优先，Pack 与来源为并列钻取维度，周报作为纵深区域
- 交互目标：把“过滤结果”和“阅读建议”直接展示在卡片层，不让用户靠猜
- 技术约束：基于现有 React + Tailwind 前端演进，不引入重型设计系统

页面结构固定为 4 个一级视图：

1. 日报首页
2. Pack 视图
3. 来源视图
4. 周报视图

## Key Changes

### 1. 整体视觉系统

采用“现代杂志 + 研究简报”的混合语言，避免当前通用 dashboard 风格。

视觉决策：

- 字体组合
  - 标题：高辨识衬线或窄体展示字，用于封面标题、栏目名、数字指标
  - 正文：耐读的人文无衬线，用于摘要、解释、元信息
- 色彩方向
  - 底色：暖白或纸张灰，不用纯白
  - 主色：深墨黑/煤灰
  - 强调色：锈红或深蓝绿，负责“已筛选信号”和交互高亮
  - 状态色：保留/存疑/过滤分别用稳定的编辑标记色，而不是大面积按钮色
- 版式
  - 桌面端使用“不对称报刊网格”
  - 移动端改为单列纵向阅读流
  - 背景加入轻微纸张纹理/噪点，头图区允许更强的视觉层次
- 动效
  - 只做 3 类：首屏渐入、栏目错峰出现、卡片 hover 展开解释
  - 不做花哨持续动画，保持阅读氛围

### 2. 日报首页

这是默认入口，也是最需要重构的页面。

首屏布局：

- 顶部 Header
  - 产品名、副标题、当前日期、时间窗口切换
  - 右侧保留全局搜索和“视图切换”入口
- Hero 封面区
  - 左侧：今日标题，例如“今天最值得读的 5 条信号”
  - 中部：一段编辑导语，说明今日总体判断
  - 右侧：三枚摘要指标
    - 已保留条目数
    - 已过滤噪音数
    - 高价值来源数
- Lead Story 主卡
  - 只放 1 条最值得读内容
  - 显示标题、来源、时间、1 句总结、为什么值得看、建议怎么读
- Top Signals 栏目
  - 4-6 条“必须看”
  - 卡片采用竖向紧凑样式，突出保留理由和价值标签
- Scan Brief 栏目
  - 6-10 条“可略读”
  - 用更轻量的条目样式，减少卡片重量
- Save For Later 栏目
  - 展示收藏型或辅助型内容，尤其承接 `x_bookmarks`
- Sidebar 式侧栏
  - 今日来源分布
  - 今日主题簇
  - 周报预告入口

卡片信息固定字段：

- 标题
- 来源名与来源类型
- 时间
- `keepDecision`
- 一句话总结
- 为什么推荐
- 可带来什么启发
- 建议动作：立即读 / 稍后读 / 只收录

禁止继续沿用当前卡片上的“纯 score + 三条分数条”作为主视觉；分数可保留到次级层，用于解释面板或详情抽屉。

### 3. Pack 视图

Pack 页的职责不是简单过滤列表，而是回答“这个主题包今天/本周值得关注什么”。

页面结构：

- Pack 顶部封面
  - Pack 名称、简介、关键词、来源数
  - 一句策略说明，例如“该 Pack 对开放流采用强过滤，对收藏流仅做阅读辅助”
- 主内容分为 3 栏目
  - 今日重点
  - 本周持续话题
  - 来源分布
- 条目排序优先展示
  - 与主题强相关
  - 高价值
  - 可行动
- 页面内支持“仅看保留内容 / 查看全部 / 查看已过滤原因”的切换

这个页面需要让用户感知 Pack 是“策展单元”，不是仅仅一组 source checkbox。

### 4. 来源视图

来源页是统一策略框架在 UI 上最直观的落点，要让用户一眼看懂这个来源被如何对待。

页面结构：

- 来源头部
  - 来源名、类型、所属 Pack、最近更新时间
  - 当前策略模式：`assist_only` 或 `filter_then_assist`
  - 来源定位文案，例如“书签型高信号流，不参与过滤”
- 关键统计
  - 今日抓取数
  - 保留率
  - 高价值率
  - 最近 7 天稳定性
- 内容区按策略分支显示
  - `assist_only`
    - 默认展示全部条目
    - 每条突出一句话总结、启发、阅读建议
  - `filter_then_assist`
    - 默认只展示保留条目
    - 提供“查看被过滤内容与原因”的折叠区
- 附加模块
  - 常见主题
  - 高价值作者/账号
  - 推荐纳入日报的比例

X 来源具体呈现规则：

- `x_bookmarks`
  - 页面标题语义为“今日书签导读”
  - 不出现“过滤掉了多少”作为主叙事
- `x_home`
  - 页面标题语义为“今日时间线筛选”
  - 必须展示保留比例与过滤理由
- `x_list`
  - 页面标题语义为“专题信号流”
  - 同时强调主题密度和可跟踪价值

### 5. 周报视图

周报不是把日报拼起来，而是“编辑部回顾”。

页面结构：

- 周报封面
  - 本周标题
  - 一段总述
  - 三个主题主张
- Repeated Signals
  - 反复出现的话题
- Worth Understanding
  - 值得系统理解而不是只扫过的内容
- Source Watch
  - 本周最有产出的来源
- Unresolved Questions
  - 本周值得继续跟踪的问题

周报卡片比日报更偏主题聚合，而不是单条内容罗列。单条内容作为主题下的证据链存在。

### 6. 导航与交互框架

整体导航从“左侧筛选栏”重构为“顶部视图导航 + 可折叠次级筛选”。

导航结构：

- 一级导航
  - 日报
  - Pack
  - 来源
  - 周报
- 二级筛选条
  - 时间窗口
  - Pack 选择
  - 来源选择
  - 搜索
  - 是否显示已过滤内容
- 桌面端
  - 一级导航放顶部
  - 次级筛选作为吸顶控制条
- 移动端
  - 一级导航用可滑动 tab
  - 次级筛选收进 drawer / bottom sheet

需要废弃的现状：

- 常驻左侧大筛选栏
- 单一流水式列表首页
- 只靠 checkbox 体现信息组织方式

### 7. 组件体系与公共接口

前端需重组为“页面组件 + 信息解释组件”两层，而不是当前通用列表组件拼装。

核心新组件：

- `EditorialShell`
- `TopNav`
- `DailyHero`
- `LeadStoryCard`
- `SignalCard`
- `ReasonBadge`
- `DecisionRibbon`
- `SourceProfilePanel`
- `ThemeClusterPanel`
- `WeeklyThemeCard`
- `FilterDrawer`

前端依赖的新增接口字段：

- 条目级
  - `keepDecision`
  - `keepReason`
  - `readerBenefit`
  - `readingHint`
  - `relevanceLabel`
  - `valueLabel`
  - `actionabilityLabel`
  - `sourcePolicyMode`
- 来源级
  - `sourcePolicyMode`
  - `sourceIntent`
  - `keepRate`
  - `dailyRecommendedCount`
- 视图级
  - `dailySummary`
  - `weeklyThemeSummary`
  - `topSignals`
  - `sourceHighlights`

建议新增独立视图接口，而不是继续让前端直接消费 `/api/items` 的平铺结果来拼首页。

## Test Plan

设计与实现阶段需要验证这些场景：

- 日报首页首屏无需滚动就能看懂“今日重点 + 推荐原因 + 下一步入口”
- `x_bookmarks` 卡片默认展示阅读建议，而非过滤标签
- `x_home` 卡片能直观看到“为何被保留”
- Pack 页与来源页对同一内容的解释字段一致，不出现语义冲突
- 周报页能展示主题聚合，而不是普通列表换标题
- 桌面端导航清晰，移动端不会因模块过多失控
- 当 AI 解释字段缺失时，UI 能平稳回退到基础摘要和来源信息
- 当没有来源/没有结果时，空状态仍保留编辑部风格，而不是裸提示框
- 颜色、字体、对比度满足基础可读性，不因风格化破坏阅读

## Assumptions

- 本轮只产出 Web 改造设计方案，不直接编码
- 首页默认仍使用“日报”作为主要视图
- 现有 React + Tailwind 栈继续沿用，视觉升级通过布局、样式系统和新组件完成
- 首期不做复杂可视化图表，优先把阅读结构、解释层和页面节奏做好
- 若进入实现，推荐拆成 3 个执行包：
  - 视图与接口层重组
  - 日报/Pack/来源页面重做
  - 周报页与响应式收尾
