- 现有 `SourcePack` 已有 `promptTemplate`/`viewTemplate` 字段，可直接扩展 `policy` 字段
- `AiEnrichmentResult` 结构清晰，可直接扩展 `keepDecision`/`keepReason`/`readerBenefit`/`readingHint`
- 现有 API 端点（`/api/items`、`/api/packs`）结构良好，扩展字段不影响向后兼容
- X 场景已有 7 种 mode（x_home, x_list, x_bookmarks 等），通过 configJson.birdMode 区分
- 前端单一卡片视图需重构为多视图路由系统
- 视图层（`src/views/`）仅用于 CLI 输出，需与 Web UI 集成
- 用户已确认：周报固定 7 天窗口、Save For Later 用后端存储、默认策略 filter_then_assist
- **设计阶段洞察**: Pipeline 应保持确定性，policy_filter 在 enrich 前执行， AI 判断结果存储到 enrichment_results 表实现无限期缓存
- **架构约束**: Hono API 路由扩展方式清晰，新路由通过 `app.route()` 注册；前端需引入 react-router-dom 支持多视图
- **Recharts 集成**: Tree-shakable，仅引入 PieChart/Tooltip 等必要组件，控制包体积
- **迁移策略**: 新建 `saved_items` 表，扩展 `enrichment_results` 添加 `filter_judgment_json` 字段， 启动时自动迁移
- **任务规划洞察**: POC 阶段共 46 个任务，覆盖类型定义、配置解析、数据库迁移、AI prompt、API 扩展、前端路由和页面组件
- **并行任务识别**: 类型定义任务（1.1-1.3）可并行执行；卡片组件（1.33-1.36）可并行；视图模块（1.38-1.42）可并行
- **验证工具**: `bun run check` 用于类型检查， `bun test` 用于测试, `bun run lint` 用于代码检查
- **E2E 验证**: `cd frontend && bun run test:e2e -- --workers=1` 可自动拉起后端 (3000) 与前端 (5173) 并完成页面级回归
- **依赖顺序**: policy.ts 类型 → load-pack.ts 解析 → policy-filter.ts pipeline → views.ts API → DailyBriefPage.tsx 前端
